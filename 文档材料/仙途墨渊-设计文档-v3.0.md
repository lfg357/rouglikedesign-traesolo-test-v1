# 第一部分：设计基础

## §1 设计支柱

四大支柱是项目的仲裁标尺，任何系统设计冲突都回到这四条来权衡。

### 1.1 四大支柱

| 支柱 | 内涵 | 落地锚点 |
|---|---|---|
| **一·落墨即杀** | 打击感是生命线。单次命中必须有"重量" | 顿帧/震屏/水墨粒子/三段连击/瞬移闪避无敌帧 |
| **二·五行随心** | 五行反应是构筑核心，"我安排的局" | 混合方案：触发用集合（低门槛），放大用五行盘邻接（高天花板） |
| **三·每一局都是独特修行** | Roguelike 随机构筑，每局 build 不同 | 反应配方扩展框架 + 角色多条 build 路径 + 遗物组合 |
| **四·留白之境** | 信息清晰不堆砌，水墨留白哲学 | 探索式地图 + UI 化解认知负荷 + 内容量宁缺毋滥 |

### 1.2 仲裁规则（冲突时按此取舍）

- **打击感 vs 系统复杂度** → 打击感优先（若某机制损害打击感，简化机制）
- **五行深度 vs 留白** → 五行深度优先，但必须用 UI 化解认知负荷（不靠阉割机制）
- **随机性 vs 公平性** → 随机性优先，但保证 build 可行性（不掉死局）
- **内容量 vs 留白** → 留白优先（宁缺毋滥，每件内容必须有存在理由）

### 1.3 支柱二的关键修正（v3.0）

v1.1 旧文档错误声称"Path B 未建/相克非机制"。代码现实是：

## §2 核心循环

### 2.1 三层循环

| 层级 | 时长 | 内容 | 验证标准 |
|---|---|---|---|
| **Moment** | 0–30s | 命中→顿帧→水墨→震屏→连击 | 顿帧时长/震屏强度/连击窗口可量化；玩家访谈"有重量" |
| **Session** | 25–45min | 清一层 5×6 迷宫 → Boss → 进层结算 | 单局时长分布、清房率（目标 60–70% 不全清）、死亡节点分布 |
| **Long-term** | 跨局 | 业力 meta 永久解锁 | 单角色 build 多样性、业力获取速率、回流率 |

### 2.2 单局时长收敛设计

5×6 网格每层 8–15 战斗房 × 30–60s/房 = 4–15min/层。5 层 = 20–75min，需收敛到 25–45min。

**收敛手段**：
1. **分支取舍**：迷宫连通性规则保证主线可达，支路可选可不选
2. **层内推深压力**：HP 跨层不回满（见 §15），逼玩家放弃支路保命
3. **单房时长上限**：战斗 30–60s，超时强制清场或惩罚
4. **房间清完即锁**：不可回退，逼取舍

### 2.3 Session Loop 闭环

```
进入层 → 探索迷宫(战斗/事件/商店) → 击败层 Boss → 层间结算屏 → 下一层
                                                ↑
                                          HP 部分保留
                                          灵气清零
                                          灵石结转
                                          遗物保留
                                          祭坛buff清零
```

## §6 五行反应系统 ★组合深度核心

### 6.1 代码现状

- **待修**：`evaluate()` 全仓零调用、`_on_board_changed` 是 pass 桩、buff 增量更新不对称

### 6.2 混合方案

- **触发层（集合式，低门槛）**：装备元素遗物 → `get_active_elements()` 集合判定 → 触发反应
- **放大层（邻接式，高天花板）**：五行盘摆放 → `calc_slot_multiplier()` 相生×1.5 / 相克×0.5 → 放大反应 stats

### 6.3 14 反应分类

| 类型 | 数量 | 触发 | 示例 |
|---|---|---|---|
| tag_count（流派堆叠） | 6 | 同 tag 遗物 ≥3 件 | 剑修→攻速 / 符修→范围 |
| element_pair（元素相生） | 2 | 双元素同时激活 | 金水→暴击 / 木火→燃烧 |
| earth_anchor（土定乾坤） | 1 | 土 + 任意元素 | 减伤 |
| element_all（五行圆满） | 1 | 金木水火土全激活 | 全属性+ |
| role_specific（角色专属） | 4 | 同角色 tag 遗物 ≥3 件 | 每角色 1 条 |

### 6.4 可扩展配方框架 ★

```gdscript
# 反应数据结构
{
    id: String,
    type: enum {tag_count, element_pair, earth_anchor, element_all, role_specific, custom},
    trigger: {
        tags: Array[String],      # 需要的 synergy_tags
        elements: Array[String],  # 需要的元素
        role: String,             # 角色专属
        min_count: int            # 最小件数
    },
    condition: Callable,          # 可选额外条件（如 HP<30%）
    stats: Dictionary,            # 生效属性
    tier: int,                    # 阶梯等级
    board_boost: bool             # 是否受五行盘放大
}
```

- 新增反应只需 `reactions.append(...)`，无需改 `evaluate` 逻辑
- **P0**：14 条 → **P1**：扩展到 24 条 → **P2**：开放反应配方 mod 式扩展

### 14.1 代码现状

### 14.2 事件池分类（5 类，P1 做 20 种）

| 类型 | 特征 | 风险收益 | 示例 |
|---|---|---|---|
| 赌徒事件 | 高风险高回报 | 期望值略负（赌徒税） | 花 30 灵石 50% 赌紫遗物 / 花 20 HP 换 50 灵气 |
| 苦修事件 | 牺牲换强化 | 确定收益有代价 | 封印 1 件遗物换永久 +10% 伤害 / 降 20 HP 上限换 +15% 攻速 |
| 奇遇事件 | 免费收益 | 纯收益但 weight 低 | 捡到蓝遗物 / NPC 赠送灵石 / 发现隐藏祭坛 |
| 陷阱事件 | 负面遭遇 | 纯负面但可规避 | 假宝箱（受伤）/ 传送门（随机位移）/ 诅咒（暂时禁用某反应） |

### 14.3 触发规则

- 进入 EVENT 房 → 从池中按 weight 随机抽 1 个事件
- 稀有事件 weight 低（奇遇 weight 5 / 普通 weight 20）
- 同一局内不重复同一事件
- 每层 EVENT 房数量固定（2–3 间）

### 14.4 风险收益曲线

- 赌徒：50% 赚 / 50% 亏，长期玩期望略负，但单次有惊喜
- 苦修：确定收益，代价明确，适合 build 确定后的强化
- 奇遇：纯收益，调节运气低谷，提升体验下限
- 陷阱：可规避（操作/知识），新手吃亏老手无伤
- 交易：等价交换，货币转换的阀门

### 14.5 分阶段

**P0** 无事件房（简化迷宫只留战斗+Boss+商店）→ **P1** 20 种事件 → **P2** 扩展

# 第三部分：长期愿景（P2/meta，方向级）

## §16 异化能力系统

### 16.1 定位

build 深度层，超越 §8 的基础 build 路径。异化能力**改变角色玩法机制**，非纯数值强化。

### 16.2 机制

- 每角色定义多条异化路径（2–3 条），每条路径是一个"形态切换"
- 激活条件：单局内达成特定条件（如"装备 3 件剑修 tag"/"激活金水反应 5 次"）
- 激活后改变角色核心机制（如燕无归"影形态"：瞬移不耗冷却但伤害降低；"狂形态"：伤害翻倍但持续掉血）
- 每角色 6 件专属遗物，分属不同异化路径，强化对应形态

### 16.3 与 §8 build 路径的关系

- §8 build 路径是"遗物+反应"的组合方向（P1 可达）
- §16 异化能力是"形态切换"的机制质变（P2 解锁）
- 两者叠加：build 路径选连击流 + 异化激活狂形态 = 高频高伤但自残的极端 build

### 16.4 解锁

P2 通过业力树解锁异化能力池，单局内靠达成条件激活。

## §17 祖物传承系统

### 17.1 定位

meta 永久装备，跨局传承感。

### 17.2 机制

- **封存**：单局内达成成就（如"无伤清一层"/"5 反应同时激活"/"单局 100 击杀"）→ 可花业力将当前关键遗物封存为祖物
- **装备**：开局带 1 件祖物（祖物槽，独立于五行盘）
- **效果**：祖物 stats 永久生效，相当于"开局送一件遗物"
- **限制**：祖物槽 1 个（防 power creep），封存有业力成本（递增）

### 17.3 设计意图

上局成果带到下局的"传承感"，让 meta 养成有具体载体而非纯数值解锁。

## §19 业力养成树

### 19.1 定位

### 19.2 树结构

7 条主线分支 + 2 条收集支线，每条分支有 3–5 个节点递进。

```
                              [起点]
    ┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
 [角色解锁线] [基线强化线] [遗物池线]  [五行盘线]  [内容池线]  [辅助线]    [挑战线]
    │           │           │           │           │           │           │
 云清子解锁   HP+10       +5通用遗物   第6槽位    +5事件类型  复活1次     难度2档
 墨尘解锁     灵石+20     +4专属遗物  阴阳槽      +1房间类型  Boss弱点   无尽模式
 慕容解锁     灵气+10     初始遗物选1 槽位乘数+0.1 +3反应配方  地图可见   挑战模式

              暴击+2%                            祭坛选项+1  事件预览   噩梦难度
```

### 19.3 七条主线明细

| 分支 | 节点（递进） | 设计意图 |
|---|---|---|
| **角色解锁线** | 云清子→墨尘→慕容清霜→异化能力池 | 解锁玩法多样性 |
| **基线强化线** | HP+10→灵石+20→灵气+10→移速+5%→暴击+2% | 小幅永久强化，不破坏平衡 |
