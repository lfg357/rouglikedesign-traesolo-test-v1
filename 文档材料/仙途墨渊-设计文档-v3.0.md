# 第一部分：设计基础

## §1 设计支柱

四大支柱是项目的仲裁标尺，任何系统设计冲突都回到这四条来权衡。

### 1.1 四大支柱

| 支柱 | 内涵 | 落地锚点 |
|---|---|---|
| **一·落墨即杀** | 打击感是生命线。单次命中必须有"重量" | 顿帧 / 震屏 / 水墨粒子 / 三段连击 / 瞬移闪避无敌帧 |
| **二·五行随心** | 五行反应是构筑核心，"我安排的局" | 混合方案：触发用集合（低门槛），放大用五行盘邻接（高天花板） |
| **三·每一局都是独特修行** | Roguelike 随机构筑，每局 build 不同 | 反应配方扩展框架 + 角色多条 build 路径 + 遗物组合 |
| **四·留白之境** | 信息清晰不堆砌，水墨留白哲学 | 探索式地图 + UI 化解认知负荷 + 内容量宁缺毋滥 |

### 1.2 仲裁规则（冲突时按此取舍）

- **打击感 vs 系统复杂度** → 打击感优先（若某机制损害打击感，简化机制）
- **五行深度 vs 留白** → 五行深度优先，但必须用 UI 化解认知负荷（不靠阉割机制）
- **随机性 vs 公平性** → 随机性优先，但保证 build 可行性（不掉死局）
- **内容量 vs 留白** → 留白优先（宁缺毋滥，每件内容必须有存在理由）

### 1.3 支柱二的核心设计

**混合方案**是五行系统的核心架构，分为两层：

- **触发层（集合式，低门槛）**：装备元素遗物 → 集合判定 → 触发反应
- **放大层（邻接式，高天花板）**：五行盘摆放 → 相生 ×1.5 / 相克 ×0.5 → 放大反应数值

两层天然难度分层——新手装备即触发，老手摆位追极致。这是支柱二"五行随心"的具体落地：入门简单（只要凑元素就有反应），精通有深度（摆位策略影响放大倍数）。

## §2 核心循环

### 2.1 三层循环

| 层级 | 时长 | 内容 | 验证标准 |
|---|---|---|---|
| **Moment** | 0–30s | 命中 → 顿帧 → 水墨 → 震屏 → 连击 | 顿帧时长 / 震屏强度 / 连击窗口可量化；玩家访谈"有重量" |
| **Session** | 25–45min | 清一层 5×6 迷宫 → Boss → 进层结算 | 单局时长分布、清房率（目标 60–70% 不全清）、死亡节点分布 |
| **Long-term** | 跨局 | 业力 meta 永久解锁 | 单角色 build 多样性、业力获取速率、回流率 |

### 2.2 单局时长收敛设计

5×6 网格每层 8–15 战斗房 × 30–60s/房 = 4–15min/层。5 层 = 20–75min，需收敛到 25–45min。

**收敛手段**：
1. **分支取舍**：迷宫连通性规则保证主线可达，支路可选可不选
2. **层内推深压力**：HP 跨层不回满（见 §15），逼玩家放弃支路保命
3. **单房时长上限**：战斗 30–60s，超时强制清场或惩罚
4. **房间清完即锁**：不可回退，逼取舍

### 2.3 Session Loop 闭环

```
进入层 → 探索迷宫(战斗/事件/商店) → 击败层 Boss → 层间结算屏 → 下一层
                                                ↑
                                          HP 部分保留
                                          灵气清零
                                          灵石结转
                                          遗物保留
                                          祭坛buff清零
```

### 2.4 从零实施指南（核心循环）

如果你是从零开始搭建这个项目，核心循环的实现顺序建议如下：

| 步骤 | 内容 | 交付物 | 验证方式 |
|---|---|---|---|
| Step 1 | 玩家角色移动 + 基础攻击 | 角色可在场景中自由移动并挥剑 | 手感测试：移动是否顺滑、攻击是否有反馈 |
| Step 2 | 单个敌人 AI + 战斗 | 敌人会追击玩家，双方可互伤 | 战斗可进行、可死亡、有胜负 |
| Step 3 | 房间制战斗 | 进入房间 → 关门 → 刷怪 → 清完开门 | 单房战斗流程完整 |
| Step 4 | 迷宫地图 + 房间切换 | 5×6 网格地图，可在房间间移动 | 探索体验成立 |
| Step 5 | 层 Boss + 层间结算 | 击败 Boss 后进下一层 | Session 循环闭环 |
| Step 6 | 遗物 + 反应系统 | 构筑深度接入循环 | 每局 build 不同的体验成立 |

详细实施任务清单见附录 F。

## §3 项目分层路线图

### 3.1 三层定义

| 层级 | 范围 | 交付物 | playtest 闸口 |
|---|---|---|---|
| **P0 核心层** | 燕无归单人 + 5×6 迷宫(战斗+Boss) + 14 反应 + 五行盘 + 14 遗物 + 灵气/灵石 + 5 种敌人 + 1 Boss + 打击感全链路 | 可玩打击感原型 | 支柱一是否成立（打击感够不够爽） |
| **P1 扩展层** | 四角色全 + 25 敌人 + 5 Boss + 40 遗物 + 20 事件房 + 商店 UI + 层间结算屏 + 5 层区域主题 + 业力基础 | 完整核心循环 | 支柱二/三是否成立（构筑深度够不够、每局是否不同） |
| **P2/meta 深度层** | 异化能力 + 祖物传承 + 黑市/祭坛 + 业力养成树 + 6 稀有度 + 反应扩展配方 | meta 养成深度 | 长期回流率是否达标 |

### 3.2 闸口规则

每层必须 playtest 验证通过才进入下一层：

- **P0 闸口**：10 名测试者中 ≥7 人认为"打击感有重量"，且单房战斗体验流畅
- **P1 闸口**：10 名测试者中 ≥7 人认为"每局 build 都不一样"，且能玩满完整 5 层
- **P2 闸口**：留存数据达标（7 日留存 ≥20%），且玩家主动追求 meta 解锁

P0 失败则重设计打击感，不急着加内容。这是支柱一"落墨即杀"的硬性要求——打击感是生命线，打击感不行一切免谈。

### 3.3 P0 阶段重点说明

P0 是整个项目的**地基阶段**，目标是验证"打击感是否成立"。这一阶段不要追求内容量，所有精力都放在手感和反馈上：

**P0 必须打磨到位的内容**：
1. 玩家移动手感（加速度、减速度、最大速度）
2. 攻击手感（前摇、命中帧、后摇、连击窗口）
3. 命中反馈（顿帧、震屏、粒子、伤害数字）
4. 闪避手感（无敌帧、位移距离、冷却时间）
5. 敌人受击反馈（停顿、击退、变色）

**P0 可以简化/省略的内容**：
1. 迷宫复杂度（可以只有一条直线）
2. 遗物数量（够用就行，不求丰富）
3. UI 美观度（功能优先，美观后补）
4. 音效（可用占位音效）
5. 多种敌人类型（1–2 种验证战斗即可）

详细 P0 实施任务清单见附录 F.1。

# 第二部分：核心系统规格（P0 + P1，规格级）

## §4 迷宫系统

### 4.1 设计规格

| 项 | 规格 | 理由 |
|---|---|---|
| 网格 | 5×6 固定，30 格上限 | 控制单局时长在 25–45min |
| 房间填充率 | 实际生成 15–20 房（非全填） | 留白 + 控制时长 |
| 战斗房占比 | 55% → 每层 8–12 间战斗房 | 收敛到 25–45min |
| 节点类型 | START / MONSTER / ELITE / SHOP / EVENT / BOSS | 标准 Roguelike 节点类型 |
| 连通拓扑 | 树状为主 + 少量环 | 保证主线可达 + 有限选择 |
| 地图可见性 | 探索式（已访问清晰，未访问模糊） | 支柱四留白 |
| 房间回退 | 清完即锁（不可回退） | 逼取舍，时长收敛关键 |
| 房间清理 | 击杀所有敌人后解锁出口 | 标准战斗房设计 |

### 4.2 节点类型说明

| 节点类型 | 功能 | 占比 | P0? |
|---|---|---|---|
| START | 出生点，无敌人 | 1 个/层 | ✓ |
| MONSTER | 普通战斗房，小怪 2–4 只 | 55% | ✓ |
| ELITE | 精英战斗房，精英怪 1 只 + 小怪 | 12% | ✓ |
| SHOP | 商店房（卦摊） | 8% | ✓（P0 简化版） |
| EVENT | 事件房 | 15% | P1 实装 |
| BOSS | 层 Boss 房 | 1 个/层 | ✓ |

### 4.3 生成算法要求

1. **连通性保证**：生成结果必须保证 START 到 BOSS 有至少一条通路
2. **分支有限**：支路数量控制在 2–3 条，不要迷宫过于复杂
3. **分布均匀**：SHOP/EVENT/ELITE 均匀分布，不要集中在某一段
4. **Boss 位置**：Boss 房必须在最后一排（第 6 列）
5. **可复现**：支持 seed 随机，同一 seed 生成相同迷宫（用于调试和分享）

### 4.4 5 层区域主题（P1）

| 层 | 区域主题 | 元素倾向 | 视觉 |
|---|---|---|---|
| 1 | 金域·锐锋 | 金属性敌人多 | 银白 / 锐利线条 |
| 2 | 木域·缠生 | 木属性敌人多 | 翠绿 / 藤蔓缠绕 |
| 3 | 水域·潜流 | 水属性敌人多 | 墨蓝 / 流动笔触 |
| 4 | 火域·焚天 | 火属性敌人多 | 朱红 / 烈焰扩散 |
| 5 | 土域·镇渊 | 土属性敌人多 | 赭黄 / 厚重块面 |

区域主题影响敌人类型 + 反应倾向 + 视觉风格，非纯数值递增。每层的敌人池、遗物池、事件池都有区域倾向，让玩家在不同层有不同的构筑策略。

### 4.5 从零实施指南（迷宫系统）

**P0 阶段实施步骤**：

| 步骤 | 任务 | 交付标准 |
|---|---|---|
| Step 1 | 定义节点类型枚举 + 房间数据结构 | 代码中可表示 6 种节点类型和房间状态（未探索/已探索/已清理） |
| Step 2 | 实现 5×6 网格迷宫生成算法 | 生成结果 START 到 BOSS 连通，MONSTER 占比 ≈55% |
| Step 3 | 实现地图 UI（小地图 / 大地图） | 玩家能看到自己位置、已探索房间、未探索房间的模糊轮廓 |
| Step 4 | 实现房间切换逻辑 | 走到出口 → 黑屏/过场 → 进入下一个房间 |
| Step 5 | 实现房间内容生成（战斗房刷怪） | 进入 MONSTER 房自动刷对应数量敌人 |
| Step 6 | 连通性 + BFS 通路验证 | 自动化测试：1000 次随机生成全部连通 |

- EVENT 房实装（见 §14）

## §5 战斗系统

### 5.1 打击感量化规格

| 反馈类型 | 普攻 | 暴击 | 处决 |
|---|---|---|---|
| 顿帧时长 | 60ms | 120ms | 200ms |
| 震屏强度 | 3.0 | 8.0 | 5.0 |
| 震屏时长 | 150ms | 200ms | 300ms |
| 水墨粒子数 | 8–10 | 12–15 | 20+ |
| 伤害数字 | 普通 | 放大 1.5× + 描边 | 全屏墨溅 |

> **实施要点**：顿帧必须用**独立于游戏时间的计时器**实现（不能用 time_scale 缩放 delta，否则会导致顿帧被无限放大）。震屏用相机偏移实现，粒子用水墨飞溅效果。

### 5.2 伤害链规范

统一伤害接口设计：

```
函数签名：
  take_damage(amount: float, source: 实体引用, is_crit: bool = false) -> void

调用链：
  攻击判定区域命中 → 获取被命中的实体 → 调用其 take_damage() → 触发受击反馈
```

**设计要求**：
- 所有可受伤实体（玩家 / 敌人 / 可破坏物）都实现此接口
- 命中判定区域与实体的关联必须清晰（通过 owner 引用或父子关系获取）
- `source` 参数用于记录伤害来源（用于反击、吸血、统计等）

### 5.3 状态机设计

使用有限状态机（FSM）管理角色行为，替代布尔分支：

```
IDLE → MOVE → ATTACK(1/2/3段) → DODGE → HURT → DEAD
```

**状态切换规则**：
- IDLE ↔ MOVE：输入方向即切换
- MOVE → ATTACK：攻击输入，有连击窗口则进入下一段
- ANY → DODGE：闪避输入，有冷却且非硬直
- ANY → HURT：受到伤害，触发受击硬直 + 无敌帧
- ANY → DEAD：HP 归零

**关键参数**：
- 连击窗口：2.5s（超过则重新从第一段开始）
- 三段连击：第三段带突进（持续 8–12 帧）
- 闪避：瞬移 + 无敌帧 0.5s + 冷却（独立变量）
- 受击：无敌帧 0.3s（独立于闪避）

### 5.4 攻击判定设计

| 项 | 规格 |
|---|---|
| 攻击类型 | 近战扇形 / 远程弹幕 / 范围领域 |
| 判定方式 | 区域重叠检测（Area2D 或等效机制） |
| 命中检测频率 | 每攻击动作检测一次（不要每帧检测） |
| 多目标 | 支持同时命中多个敌人 |
| 命中反馈 | 顿帧 + 震屏 + 粒子 + 伤害数字 + 击退 |

### 5.5 从零实施指南（战斗系统）

**P0 阶段实施步骤**：

| 步骤 | 任务 | 交付标准 |
|---|---|---|
| Step 1 | 实现玩家移动 + 基础攻击动画 | 角色可移动、可挥剑、动画播放正确 |
| Step 2 | 实现攻击判定区域（hitbox） | 挥剑时生成判定区域，命中敌人有响应 |
| Step 3 | 实现 take_damage 接口 + HP 系统 | 玩家和敌人都有 HP，受击扣血，归零死亡 |
| Step 4 | 实现命中反馈链：顿帧 + 震屏 + 粒子 + 伤害数字 | 命中有明显"重量感"，playtest 验证 |
| Step 5 | 实现三段连击 + 连击窗口 | 连续攻击进入二段、三段，第三段有突进 |
| Step 6 | 实现闪避 + 无敌帧 | 闪避有位移、有 0.5s 无敌、有冷却 |
| Step 7 | 实现受击硬直 + 受击无敌帧 | 受击有短暂停顿 + 0.3s 无敌，防止连击秒杀 |
| Step 8 | 打击感调优（反复迭代） | 10 人 playtest ≥7 人认为"有重量" |

**P1 阶段扩展**：
- 多种攻击类型（远程 / 召唤 / 领域）
- 暴击系统
- 元素附魔攻击
- 更多受击反馈类型

## §6 五行反应系统 ★组合深度核心

### 6.1 混合方案

- **触发层（集合式，低门槛）**：装备元素遗物 → 集合判定 → 触发反应
- **放大层（邻接式，高天花板）**：五行盘摆放 → 相生 ×1.5 / 相克 ×0.5 → 放大反应数值

两层天然难度分层——新手装备即触发，老手摆位追极致。

### 6.2 14 反应分类（P0）

| 类型 | 数量 | 触发条件 | 示例 |
|---|---|---|---|
| tag_count（流派堆叠） | 6 | 同 tag 遗物 ≥3 件 | 剑修→攻速 / 符修→范围 |
| element_pair（元素相生） | 2 | 双元素同时激活 | 金水→暴击 / 木火→燃烧 |
| earth_anchor（土定乾坤） | 1 | 土 + 任意元素 | 减伤 |
| element_all（五行圆满） | 1 | 金木水火土全激活 | 全属性+ |
| role_specific（角色专属） | 4 | 同角色 tag 遗物 ≥3 件 | 每角色 1 条 |

**详细反应配方表见附录 D.1。**

### 6.3 可扩展配方框架 ★

反应系统采用**数据驱动**设计，新增反应只需追加配方数据，无需修改核心逻辑：

```
反应数据结构：
{
    id: 字符串（唯一标识）
    name: 字符串（显示名称）
    type: 枚举 {tag_count, element_pair, earth_anchor, element_all, role_specific, custom}
    trigger: {
        tags: 字符串数组（需要的协同标签）
        elements: 字符串数组（需要的元素）
        role: 字符串（角色专属，可选）
        min_count: 整数（最小件数）
    }
    condition: 函数引用（可选额外条件，如 HP<30%）
    stats: 字典（生效属性）
    tier: 整数（阶梯等级，用于 UI 展示稀有度）
    board_boost: 布尔（是否受五行盘放大）
}
```

**扩展路线**：
- **P0**：14 条基础反应
- **P1**：扩展到 24 条（加更多元素组合 + 隐藏反应）
- **P2**：开放反应配方扩展（mod 式或玩家解锁式）

### 6.4 系统运行机制

1. **触发时机**：遗物装备 / 移除时、五行盘变化时、角色状态变化时（如 HP 变化触发残血反应）
2. **重算逻辑**：触发时遍历所有反应配方，逐条检查条件，满足则激活
3. **数值叠加**：激活的反应将 stats 叠加到角色属性上（支持加法叠加、乘法叠加）
4. **Board Boost**：激活反应时，检查对应元素遗物在五行盘中的邻接关系，乘以放大系数

## §7 五行盘系统

### 7.1 设计规格

| 项 | 规格 |
|---|---|
| 槽位数 | 5（P0）→ 6（P2 业力树解锁）→ 7（P2 阴阳槽） |
| 布局 | 五边形，槽位按五行方位排布（金右 / 木左 / 水上 / 火下 / 土中） |
| 相生边 | 金→水→木→火→土→金，相邻相生 ×1.5 |
| 相克边 | 金→木→土→水→火→金，相邻相克 ×0.5 |
| 摆放规则 | 玩家拖拽遗物到槽位（或点击选择放置） |

### 7.2 邻接乘数计算

每个槽位的乘数 = 该槽位遗物与所有相邻槽位遗物的相生/相克关系之和：

- 如果某相邻槽位有遗物，且与本槽位遗物**相生** → 本槽乘数 +1.5
- 如果某相邻槽位有遗物，且与本槽位遗物**相克** → 本槽乘数 ×0.5
- 如果相邻槽位为空 → 不影响

总乘数 = 所有放置了元素遗物的槽位乘数的平均值（或累加，需 playtest 调优）

### 7.3 UI 规格（P1）

- 五边形 5 槽位可视化（水墨风格）
- 相生边绿色高亮 / 相克边红色高亮（实时）
- 拖拽放置 + 实时乘数预览（放下前显示预期 ×1.5/×0.5）
- **一键最优摆放**按钮（辅助新手，按当前遗物组合算最大乘数摆放）
- 新手引导 Beat 4：首件遗物自动放入对应槽位 + 高亮相生链

### 7.4 玩家交互流程

1. 获得元素遗物 → 进入背包 / 五行盘界面
2. 拖拽遗物到 5 槽之一
3. 实时预览乘数变化（相生边亮绿 / 相克边亮红）
4. 确认放置 → 触发板变化事件 → 反应系统重算 → stats 更新

### 7.5 从零实施指南（五行盘系统）

**P0 阶段实施步骤**：

| 步骤 | 任务 | 交付标准 |

| Step 1 | 定义五行盘数据结构：槽位数 + 每个槽位放置的遗物 | 数据层完整，可放置 / 移除 / 查询 |

- **交易**：等价交换，货币转换的阀门

│  祭坛强化：清零                         │
└─────────────────────────────────────────┘

### 15.3 进层流程（闭环）

1. Boss 死亡 → 触发层通关
2. 弹出结算屏，暂停游戏
3. 显示本层战绩 + 货币结算 + 下层预览
4. 玩家点击"继续探索" → 加载下一层迷宫
5. 灵气清零 / 灵石结转 / 祭坛 buff 清零 / 五行盘保留 / 遗物保留
6. 玩家出生在下一层 START 房

### 15.4 保留与清零规则表 ★

| 项目 | 跨层保留？ | 理由 |
|---|---|---|
| **HP** | **部分保留（不回满）** | **层间推深压力，通过掉落 / 商店 / 事件回血道具弥补** |
| 灵气 | 清零 | 层内货币设计 |
| 灵石 | 保留 | 单局货币 |
| 业力 | 保留 | 永久 |
| 遗物（背包 + 五行盘） | 保留 | build 积累 |
| 祭坛 buff | 清零 | 层内临时 |
| 反应激活状态 | 保留（自动重算） | build 一致性 |
| 槽位扩展 | 保留 | 永久升级 |

> **HP 不回满是关键设计**：这是时长收敛和层间压力的核心机制。如果每层都满血，玩家会倾向于清完所有房间，单局时长会失控。HP 不回满让玩家必须做取舍——是继续探路拿奖励还是直接去 Boss 保命。

### 15.5 HP 回复阀门设计

| 渠道 | 回血量 | 触发方式 |

### 15.6 死局保护

HP 低于 max_hp 的 15% 时进入下一层会自动补到 15%，避免硬死局。Boss 房前保底回血 30% 保证 Boss 战始终可打。

# 第三部分：长期愿景（P2/meta，方向级）

## §16 异化能力系统

### 16.1 定位

build 深度层，超越 §8 的基础 build 路径。异化能力**改变角色玩法机制**，非纯数值强化。

### 16.2 机制

- 每角色定义多条异化路径（2–3 条），每条路径是一个"形态切换"
- 激活条件：单局内达成特定条件（如"装备 3 件剑修 tag" / "激活金水反应 5 次"）
- 激活后改变角色核心机制（如燕无归"影形态"：瞬移不耗冷却但伤害降低；"狂形态"：伤害翻倍但持续掉血）
- 每角色 6 件专属遗物，分属不同异化路径，强化对应形态

### 16.3 与 §8 build 路径的关系

- §8 build 路径是"遗物 + 反应"的组合方向（P1 可达）
- §16 异化能力是"形态切换"的机制质变（P2 解锁）
- 两者叠加：build 路径选连击流 + 异化激活狂形态 = 高频高伤但自残的极端 build

### 16.4 解锁

P2 通过业力树解锁异化能力池，单局内靠达成条件激活。

## §17 祖物传承系统

### 17.1 定位

meta 永久装备，跨局传承感。

### 17.2 机制

- **封存**：单局内达成成就（如"无伤清一层" / "5 反应同时激活" / "单局 100 击杀"）→ 可花业力将当前关键遗物封存为祖物
- **装备**：开局带 1 件祖物（祖物槽，独立于五行盘）
- **效果**：祖物属性永久生效，相当于"开局送一件遗物"
- **限制**：祖物槽 1 个（防止数值膨胀），封存有业力成本（递增）

### 17.3 设计意图

上局成果带到下一局的"传承感"，让 meta 养成有具体载体而非纯数值解锁。

## §18 黑市与五行祭坛

### 18.1 定位

高风险高回报事件，build 博弈的深度阀门。

### 18.2 黑市（特殊商店）

- 出现：低概率替换普通商店（5–10%）
- 商品：稀有遗物（紫 / 金）+ 祖物碎片 + 反应配方解锁
- 价格：高（灵石 + 业力双重消耗）
- 风险：商品随机，可能买到不契合 build 的遗物

### 18.3 五行祭坛（消耗换强化）

- 机制：消耗 1 件遗物 → 换取随机强化（永久本局）
- 强化池：属性大幅提升 / 解锁隐藏反应 / 槽位 +1
- 风险：有 30% 概率"反噬"（强化失败，遗物白消耗，且扣 HP）
- 与五行盘联动：消耗的遗物元素影响强化倾向

### 18.4 与节点类型的关系

作为 EVENT 房的稀有子类型出现。

## §19 业力养成树

### 19.1 定位

meta 解锁体系总纲，§16 / §17 / §18 的入口。

### 19.2 树结构

7 条主线分支 + 2 条收集支线，每条分支有 3–5 个节点递进。

```
                              [起点]
    ┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
 [角色解锁线] [基线强化线] [遗物池线]  [五行盘线]  [内容池线]  [辅助线]    [挑战线]
    │           │           │           │           │           │           │
 云清子解锁   HP+10       +5通用遗物   第6槽位    +5事件类型  复活1次     难度2档
 墨尘解锁     灵石+20     +4专属遗物  阴阳槽      +1房间类型  Boss弱点   无尽模式
 慕容解锁     灵气+10     初始遗物选1 槽位乘数+0.1 +3反应配方  地图可见   挑战模式
 异化能力池   移速+5%     祖物碎片掉落 五行圆满+   词缀池扩展  商店标记   祖物槽2
              暴击+2%                            祭坛选项+1  事件预览   噩梦难度
```

### 19.3 七条主线明细

| 分支 | 节点（递进） | 设计意图 |
|---|---|---|
| **角色解锁线** | 云清子 → 墨尘 → 慕容清霜 → 异化能力池 | 解锁玩法多样性 |
| **基线强化线** | HP+10 → 灵石+20 → 灵气+10 → 移速+5% → 暴击+2% | 小幅永久强化，不破坏平衡 |
| **遗物池线** | +5 通用遗物 → +4 专属遗物 → 初始遗物选 1 → 祖物碎片掉落 | 扩展 build 素材池 + 开局选择 |
| **五行盘线** | 第 6 槽位 → 阴阳槽 → 槽位乘数 +0.1 → 五行圆满强化 | **对应支柱二，深化五行盘策略** |
| **内容池线** | +5 事件类型 → +1 房间类型 → +3 反应配方 → 词缀池扩展 → 祭坛选项 +1 | **扩展单局内容丰富度** |
| **辅助线** | 复活 1 次 → Boss 弱点提示 → 地图可见度 → 商店标记 → 事件预览 | **降低挫败感，辅助新手** |
| **挑战线** | 难度 2 档 → 无尽模式 → 挑战模式 → 祖物槽 2 → 噩梦难度 | **高玩深度内容** |

### 19.4 两条收集支线

| 支线 | 解锁条件 | 效果 |
|---|---|---|
| **图鉴系统** | 首次拾取 / 首次击杀 / 首次激活 | 遗物图鉴 / 敌人图鉴 / 反应图鉴（查看已收集内容，含未解锁预览） |
| **结局分支** | 通关不同条件 | 多结局（速通结局 / 全收集结局 / 无伤结局 / 心魔结局） |

### 19.5 业力成本曲线（方向级）

| 节点类型 | 业力成本 |
|---|---|
| 基线节点 | 100–300 |
| 中阶节点 | 300–600 |
| 高阶节点（异化 / 第 6 槽 / 复活） | 800–1200 |
| 挑战线终点 | 1500+ |

目标：单局 50–200 业力，10–20 局解锁一个核心节点，保证 meta 养成长期有进度感但不拖沓。

### F.4 实施建议顺序

对于从零开始的团队，建议按以下优先级推进：

```
P0 核心打击感 → P0 战斗闭环 → P0 迷宫 → P0 构筑系统 → P0 经济 → P0 层间闭环
     ↓
P1 角色扩展 → P1 敌人扩展 → P1 内容填充 → P1 UI 完善 → P1 区域主题
     ↓
P2 meta 系统 → P2 内容扩展 → P2 运营优化
```

**每个阶段结束后必须做 playtest 验证，失败则回退重做，不要急于进入下一阶段。**
